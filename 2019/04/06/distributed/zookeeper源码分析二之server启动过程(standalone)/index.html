ZooKeeperServer 的 startdata 实现

startdata 加载节点树、日志信息到内存中。

通过 volatile 来保证 db 初始化状态的可见性，保证 db 只能够初始化一次。循环遍历 session 超时的节点并删除节点。源码中有一段注释信息,Leader 调用 lead 方法的时候会调用该方法，但是 db 在选主之前已经初始化,所以 server 在第一次投票的时候不能用它自己的 zxid 参与投票。具体细节在集群启动的时候在深究。

小结

至此,zookeeper 单机启动的方式已经做了大概的描述，当然内部还有很多其他的细节没有涉及到。

  • JDK NIO 的方式比较复杂,需要频繁设置读取位置。Netty 的实现方式比较简洁,这里也没有做具体的说明.
  • 请求接收和响应是核心以及 jute 的序列化.
  • 线程安全的保证、读写锁、volatile 保证变量对线程的可见性.
